How to use

/amplitude-guide

Or just ask a data question: "what's our signup to paid conversion?", "how are signups trending?", "which events should I use for activation analysis?"

What it does

Before running any Amplitude query, the skill asks three questions: timeframe, conversion window, and scenario-specific filters. It then guides you to the right events, warns about traps, and references pre-built curated charts as starting points.

Workflow

1 Understand the question — what analysis is the user trying to run?
2 Ask clarifying questions — timeframe, conversion window, and scenario-specific filters
3 Map to curated events — select the right events from the registry with correct filters
4 Warn about traps — flag misleading events or missing filters before querying
5 Run or guide the analysis — query via Amplitude MCP or point to curated charts

Key traps the skill catches

Unfiltered Plan changed — includes trials, free downgrades, and direct sales. Must filter by productId.

Unfiltered Sent email on server — includes plain emails with no Mixmax features. Must use positive filters (withSequence, withTemplate, etc.).

default_calendarlink_in_signature — passive, auto-added. Not a real calendar enhancement. Exclude from calendar analysis.

Virality leadSource — includes workspace invites when not filtered to isFirstWorkspaceAdmin. Segment with viralSource.

Onboarding events as product usage — Onboarding_Step_Shown, Created profile, etc. are instrumentation, not activation signals.

Rate without volume — conversion rate can be misleading when traffic mix shifts. Always check absolute numbers alongside rates.
